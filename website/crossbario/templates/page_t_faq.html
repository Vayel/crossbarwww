Frequently Asked Questions

What is the relationship between WAMP, Autobahn and Crossbar.io?
  • WAMP is an open source protocol which provides RPC and PubSub messaging patterns.
  • The Autobahn project provides WAMP implementations in multiple languages.
  • Crossbar.io builds on Autobahn|Python and provides a much larger set of functionality to make it into a powerful router for application messaging.

In more detail:

WAMP (Web Application Messaging Protocol) is a protocol which provides both Remote Procedure Calls (RPC) and Publish & Subscribe (PubSub) messaging patterns. WAMP enables distributed, multi-client and server applications. It is an open protocol, and several independent implementations exist.

The Autobahn project provides open-source WAMP implementations for several programming languages (at the moment: Python, JavaScript, C++). These are WAMP client implementations, with the exception of Autobahn|Python, which includes basic router capabilities.

Crossbar.io builds on Autobahn|Python. It adds the full set of advanced WAMP router functionality, as well as things like an integrated Web server, hosting and centralized managemeent of application components or support for various authentication methods.

Applications developed with Crossbar.io can use any WAMP client library. This, obviously, includes the Autobahn libraries.

What does Crossbar.io's open-source license mean for me when I use it for a project?

The license (AGPL v3) does not have any effect on your application.

The APGL only affects code which is joined to a AGPL licensed project, i.e. the Crossbar.io code in this case.

When using Crossbar.io for an application, your application code just sends and receives WAMP messages which Crossbar.io routes. This does not join your code to the code of Crossbar.io.

Your code remains yours, and you can license it in whichever way you want.

Can I hack Crossbar.io to fit my own needs?

Like all open source licenses, the AGPL gives you the right to modify the code: to fix bugs, optimize things, add features, or anything else you need or feel like doing.

When you modify Crossbar.io, then your modifications fall under the AGPL if

  • you distribute the modified code to a third party
  • you run the modified code on a server accessible to a third party

A 'third party' would usually be anybody else but you, except for when you are working for a company - use purely inside the company does not trigger the AGPL.

In these cases, you need to provide access to your modified code - and do so under the conditions of the AGPL.

What this means is that you cannot use Crossbar.io as the basis for developing closed source software (except for when you are its only user).

I want to contribute to Crossbar.io - what do I need to do?

There are full instructions for how to contribute.

The short version:

Crossbar.io is hosted on GitHub, so you need to be familiar with the git development model. (If not, you can start by taking this little course for free).

Additionally, before we can accept your first contribution, you need to sign a Contributor Assignment Agreement (CAA) and mail this to us.

This is needed in order for the Crossbar.io project to have all necessary rights to e.g. to switch licenses in the future.
