Features

The primary function of Crossbar.io is as an advanced WAMP router.

In addition to this, it aims to offer functionality which makes the deployment of WAMP applications easier and extends connectivity.

Below is an overview of current and planned features, with the latter marked as such.

WAMP routing

  • Implements the WAMP basic profile
  • Implements the WAMP adavanced profile Partially implemented Under development

Scalability & Federation

Other features

  • Integrated Web server to serve your Web application.

Database Connectors

Under development

Modern databases can be much more than simple data stores. They offer fully featured programming languages which are optimized for the processing of stored data.

With a database connector, we want to offer as much of the WAMP functionality as possible for code which runs directly in your database.

This means that you can move your business logic into the database, close to your data.
Using the Web server integrated into Crossbar.io, you will be able to write database applications without the need for an additional application server.

We are planning on offering database connectors for

  • PostgreSQL
  • Oracle

With PostgreSQL, you even get pluggable programming languages. For example, you can use plv8js to run JavaScript in the database.

REST integration

Under development

While WAMP is all you need to develop your own applications, Web services today use REST APIs.

With REST bridging, Crossbar.io will be able to do some translation between WAMP and REST.

Specificially you will be able to

  • publish events via HTTP POST to a REST API endpoint in Crossbar.io Under developement
  • deliver PubSub events to a REST API endpoint Under consideration
  • route a WAMP remote procedure call to a Web service REST API endpoint Under consideration
  • call a remote procedure via a REST API call to an endpoint in Crossbar.io Under consideration
